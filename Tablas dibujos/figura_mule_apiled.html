Experience APIs

Adaptan solicitudes y respuestas a las necesidades específicas de cada canal. Cada consumidor tiene su propia API adaptada sin impactar a los demás.

Process APIs

Implementan la lógica de los procesos de negocio de forma reutilizable. Un cambio en el proceso de alta afecta una sola API y todos los canales lo reciben automáticamente.

System APIs

Encapsulan el protocolo técnico de cada sistema backend (BAPI, EDIEL, REST) y los exponen con una interfaz REST homogénea hacia las capas superiores.
